Git – сервис для работы над версиями. Нужен для командной работы, чтобы работу можно было вести с разных компьютеров.
Два варианта работы в GIT: через веб-интерфейс, из терминала.
Для того, чтобы работать в GIT, необходимо:
- Зарегистрироваться на сайте GitHub
- Создать remote repository
- Загрузить/создать исходный файл, над которым планируется работа
- Создать ветку, в которой будут добавляться изменения
- Разделить обязанности между работниками, чтобы каждый мог работать в своей области
Каждый работник, кто будет вносить изменения:
- Копирует на локальный компьютер pull remote или продолжает работать онлайн в веб-версии
- Создает собственную ветку на локальном компьютере branch
- Работает в своей ветке (add, commit)
- Делает Push в свой сервер
- Делает merge (добавить свою часть в общую версию)
Основные команды для работы в GIT.
Перед началом работы в консоли необходимо представиться:
git config --global user.name "Your Name" # указать имя, которым будут подписаны коммиты
git config --global user.email "e@w.com"  # указать почту
git init создать локальный репозиторий в текущей папке
git add добавляет файлу версионность в локальном репозитории
git commit -m”” фиксирует изменения, в кавычки добавляют комментарий
git diff отличие текущего файла от сохраненного
git log выводит журнал всех изменений
git checkout помогает переключиться между коммитами
